# おしごとちょう を GitHub で公開する手順

このフォルダには、GitHub Pagesで公開するために必要なファイルが入っています。

```
index.html      ← おしごとちょう本体（これがアプリの中身）
manifest.json   ← ホーム画面に追加したときのアプリ名・アイコン設定
icon-192.png    ← アプリアイコン（小）
icon-512.png    ← アプリアイコン（大）
favicon.png     ← ブラウザタブに出る小さいアイコン
```

**4つのファイルすべてを同じ場所（リポジトリのトップ）にアップロードしてください。**
フォルダの中に入れたり、ファイル名を変えたりしないようにご注意ください。

---

## 手順1：GitHubのアカウントを作る（すでにある場合はスキップ）

1. スマホのブラウザで [https://github.com](https://github.com) を開く
2. 右上の「Sign up」をタップ
3. メールアドレス・パスワード・ユーザー名を入力して登録
   - ユーザー名はあとで公開URLの一部になります（例：`tarou123`）
4. 届いた確認コードを入力すれば完了

---

## 手順2：リポジトリ（保管フォルダ）を作る

1. 右上の「＋」→「New repository」をタップ
2. 「Repository name」に好きな名前を入力
   - 例：`oshigotocho`
3. 「Public」が選ばれていることを確認（Privateだと無料公開ができません）
4. 「Add a README file」をオンにする
5. 「Create repository」をタップ

---

## 手順3：ファイルをアップロードする

1. 作成したリポジトリのページで「Add file」→「Upload files」をタップ
2. `index.html` / `manifest.json` / `icon-192.png` / `icon-512.png` / `favicon.png` の
   **5つのファイルをすべて選んで**アップロード
3. 画面下までスクロールして「Commit changes」をタップ
   - これを押さないとアップロードが完了しません

---

## 手順4：GitHub Pagesを有効にする

1. リポジトリのページ上部「Settings」タブをタップ
2. 左側のメニューから「Pages」を選ぶ
3. 「Build and deployment」→「Source」が
   「Deploy from a branch」になっていることを確認
4. 「Branch」のところで `main` を選び、フォルダは `/ (root)` のまま
5. 「Save」をタップ

数分待つと、ページの上部に
**「Your site is live at `https://ユーザー名.github.io/リポジトリ名/`」**
という案内が表示されます。これが公開URLです。

---

## 手順5：スマホで開く・ホーム画面に追加する

1. 上記のURLをスマホのブラウザ（Chrome等）で開く
2. 正しく「おしごとちょう」が表示されることを確認
3. ブラウザのメニュー（︙）から「ホーム画面に追加」を選ぶ
4. アイコン（添付していただいた画像）付きでホーム画面に追加され、
   アプリのようにタップして起動できるようになります

---

## つまずきやすいポイント

- **画像が表示されない／アプリ名が反映されない**
  → ファイル名の大文字・小文字が一致しているか確認してください
  （`Icon-192.png` と `icon-192.png` は別のファイルとして扱われます）

- **公開してもページが真っ白**
  → アップロードした場所がリポジトリの一番上（root）になっているか確認してください。
    フォルダの中に`index.html`が入っていると見つけてもらえません。

- **更新しても変わらない**
  → 反映には数分〜最大10分ほどかかることがあります。少し待ってから再読み込みしてください。

- **マイク機能（音声入力）が動かない**
  → GitHub Pagesは自動的に `https://` で公開されるため、
    `file://` で直接開いていたときの「マイクが使えない」問題はここで解消されるはずです。

---

## データについて

このアプリのデータ（おしごとの記録・おこづかいなど）は、スマホの中の
「LocalStorage」というブラウザの保存領域に入ります。GitHubには記録データは一切送られません。

- 同じスマホ・同じブラウザで開けば、データは続きから使えます
- 別のスマホやブラウザで開くと、データは空の状態から始まります
- 引っ越したい場合は、アプリ内の「設定」→「データをエクスポート」でファイルに書き出し、
  別の端末で「データをインポート」すれば移行できます
